myFlix

The objective of this server-side apllication of to provide users with information on their favorite movies. User are able to sign up and create an account, update their personal information and list of their favorite movies.

Endpoint HTTP Method Request Format Response Format Use Case
/api/movies GET None JSON Returns list on all movies.
/api/movies/[title] GET None JSON Returns all stored information on the specified movie (title as specification)
/api/movies/[title]/genre GET None JSON Return data about a genre (description) by name/title (e.g., “Thriller”)
/api/movies/[title]/director GET None JSON Return data about a director (bio, birth year, death year) by name
/api/users POST JSON
{ 'username': '[username]', 'password': '[password]', 'firstName': '[firstName]', 'lastName': '[lastName]' 'birdthday': [birthday], 'movies': [ ] } 		Text relaying status Allow all users to register
/api/users/[username] PUT None Text relaying status Allow user to update username
/api/movies PUT JSON
{ 'title': [title], 'description': [description], 'director': { 'name': [name], 'bio': [bio], 'birdthday': [birthday], 'deathday': [deathday] } } 		Tect relaying status Allow users to add a movie to their list of favorites (showing only a text that a movie has been added—more on this later)
/api/movies DELETE JSON
{ 'title': [title] } 		Text relaying status Allow users to remove a movie from their list of favorites (showing only a text that a movie has been removed—more on this later)
/api/users DELETE JSON
{ 'username': [username] } 		Allow existing users to deregister (showing only a text that a user email has been removed—more on this later)
